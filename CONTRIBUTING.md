# Contributing to Minimal-IDS

Thank you for considering contributing to the Minimal-IDS project! We welcome contributions from students and researchers. Please follow the guidelines below to ensure smooth collaboration.

## How to Contribute
1. **Fork the Repository**: Create a personal fork of the repository to work on your changes.
2. **Create a Feature Branch**: Branch off of the `main` branch, and name your branch descriptively (e.g., `feature/add-new-algorithm`).
3. **Make Your Changes**: Implement your changes, ensuring you follow the coding style and conventions used in the project.
4. **Document Your Changes**: If you add new functionality or make changes to existing code, please include appropriate documentation.
5. **Run Tests**: Ensure that all tests pass before you submit your changes.
6. **Submit a Pull Request**: After completing your changes, submit a pull request to the `main` branch of the original repository. 

## Pull Request Guidelines
- Clearly describe the purpose of your pull request.
- Link any relevant issues or discussions.
- Provide screenshots or examples to illustrate significant changes.

## Code of Conduct
We promote a welcoming and inclusive environment for all contributors. Please be respectful and constructive in your communications. Any behavior contrary to this will not be tolerated.

## Questions?
If you're unsure about anything, feel free to reach out by opening an issue or contacting the project maintainers directly.
